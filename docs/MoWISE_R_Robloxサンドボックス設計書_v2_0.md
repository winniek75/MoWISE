## 1. コンセプト

### 1.1 コンセプト一文

> **「かつて栄えた世界は荒れ果てた。Mowiと共に荒野を開拓し、英語の力で自分だけのユートピアを築け。」**

### 1.2 プレイヤーの体験ナラティブ

プレイヤーが最初に降り立つのは、かつて美しい街があった廃墟の大地。
崩れた建物の残骸、雑草に覆われた道、誰もいなくなった世界。

そこにMowiが現れる。Mowiはプレイヤーのアバターの肩に止まり、最初の言葉をかける。

> *「ここは昔、みんなが言葉を交わしていた場所だよ。  
> もう一度、賑やかな街にできるかな。一緒にやってみよう。」*

英語パターンを使うたびに、世界が少しずつ動き出す。
土地を整地し、素材を集め、家を建て、道を引く。
やがて町民がやってきて、英語で話しかけてくる。
コミュニケーションが増えるほど街は豊かになり、農業や商売へと発展していく。

これは「勉強して報酬をもらう」ゲームではない。
**英語を使うこと自体が、この世界を動かす力**だ。

## 3. Mowiの新しい立ち位置

### 3.1 「相棒・メンター」としてのMowi

v1.0では「街の魂として上空に浮かぶ」存在だったMowiを、v2.0ではプレイヤーと**常に一緒に行動する相棒**として再定義する。

Webアプリの「内なる鏡（プレイヤーの英語力の反映）」という原則は変わらない。
Mowiの状態はプレイヤーの習熟度に連動する。ただし、表現方法が変わる。

| Webアプリ | Roblox v2.0 |
|----------|-------------|
| 画面の中で輝くオーブ | プレイヤーのアバターと並んで歩く・飛ぶ存在 |
| Lv別の彩度・明度変化 | 体の輝き・リングの速度・翼の形で状態を表現 |
| チェックイン日記に反応 | 「昨日、頑張ってたね」とゲーム内でセリフ |
| パターン★達成時にアニメーション | 「やった！伝わった！」と喜ぶリアクション |

### 3.2 Mowiの行動パターン（3D空間）

| 状況 | Mowiの行動 |
|------|-----------|
| 探索中 | プレイヤーの周囲を飛び回る。廃墟を指さして「ここ、何があったんだろうね」 |
| Flash Output前 | 「いくよ！」と身構える |
| 正解時 | 「そう！それだ！」と光が強くなる |
| 不正解時 | 「うーん、もうちょっとかな」と首を傾ける（批判しない） |
| 建物完成時 | 「見て！」と完成した建物の方向を向いて光る |
| 町民が来たとき | 「お客さんだ！話しかけてみて！」と案内役になる |
| 長時間放置後 | 「おかえり。待ってたよ」と出迎える |
| ★5達成時 | 「すごい…完璧だ。この場所、本当に変わったね」と感慨深げに |

### 3.3 Mowiはペットではない（設計ルール）

- Mowiはプレイヤーに「依存しない」。常に対等な相棒として振る舞う
- Mowiはプレイヤーを「褒めすぎない」。英語の習熟度の鏡であり続ける
- Mowiはプレイヤーに「媚びない」。時に率直に「それは違う」と言える
- Mowiの輝きはプレイヤーの意思で変えられない。英語力の結果として変化する

## 4. サンドボックス設計

### 4.1 ゲームプレイの基本構造

```
① 英語チャレンジ（Flash Output）
      ↓
② 素材・Word Coins を獲得
      ↓
③ 土地を整地する（荒れた地形を平らにする）
      ↓
④ 素材でアイテム・建物・インフラをクラフト
      ↓
⑤ 自由に配置・レイアウト
      ↓
⑥ 町民が移住してくる → 英語で会話 → アイテム・素材を交換
      ↓
⑦ 農業・商売に発展 → 街がさらに豊かになる
      ↓
⑧ 新エリア（廃墟ゾーン）が解禁 → ①に戻る
```

### 4.2 素材システム

英語チャレンジで得られる素材は、使用したパターンの「意味・文脈」と連動する。

| 英語パターン | 獲得素材 | 理由 |
|------------|---------|------|
| `I'm [名前].` | 石材・木材（基礎素材） | 自己紹介 = 存在を示す = 土台 |
| `I like [名詞].` | 花・装飾素材 | 好きなもの = 美しさ・彩り |
| `I want [名詞].` | 食料素材 | 欲しいもの = 必要なもの |
| `I have [名詞].` | 倉庫拡張素材 | 持っている = 所有・保管 |
| `I can [動詞].` | 工具素材 | できること = 能力・技術 |
| `I'd like [名詞], please.` | 高級建材 | 丁寧な表現 = 質の高い素材 |
| 長文パターン（P050〜） | レア素材・鉱石 | 複雑な表現 = 希少な材料 |

### 4.3 クラフトシステム

**整地**
- ツルハシ（基礎素材でクラフト）を使って荒れた土地を平らにする
- 岩を砕くと石材が手に入る
- 雑草を刈ると肥料が手に入る（後で畑に使える）

**クラフトカテゴリ**

| カテゴリ | 内容 |
|---------|------|
| 家具 | テーブル・椅子・ベッド・本棚・ランプ・絵 |
| 建物 | 家・学校・カフェ・病院・倉庫・工房 |
| インフラ | 道路・街灯・橋・フェンス・公園 |
| 農業 | 畑・ハウス・水路・ウェルカムゲート |
| 商業 | お店・市場の屋台・カフェ・取引所 |
| 装飾 | 花壇・旗・噴水・彫刻・看板 |

**クラフトと英語習熟度の連動**

| パターン★ | 解禁クラフト |
|-----------|------------|
| ★1 | 基本建材・道路・簡単な家具 |
| ★2 | 農業設備・お店の基礎 |
| ★3 | 装飾アイテム・複合建物 |
| ★4 | レア外観・特殊インフラ |
| ★5 | ランドマーク・ユニーク建物（世界に1つ） |

### 4.4 農業・商売システム

**農業フロー**
```
畑をクラフト → 種を植える（英語チャレンジで種獲得）
→ 毎日ログインで水をやる → 作物が育つ
→ 収穫 → 食料素材として使う or 町民に売る
```

**商売フロー**
```
お店をクラフト → 看板の商品名を英語で入力
→ 町民が来店 → 英語で接客（NPC会話ミッション）
→ 正しく応答 → 売上（Word Coins）獲得
→ 売上で素材を仕入れ → お店をアップグレード
```

## 5. 町民システム

### 5.1 町民の移住メカニズム

タイクーン方式（Coinsで建物を建てたらNPC配置）ではなく、**自然に移住してくる**設計。

```
街の発展度が一定値を超える
      ↓
Mowiが「だれかが近づいてきてる」と知らせる
      ↓
廃墟の端から一人の町民が歩いてくる
      ↓
プレイヤーが話しかけると「ここに住んでもいいですか？」
      ↓
英語で受け答え（NPC会話ミッション）
      ↓
移住確定 → 町民が街に住み始める
```

### 5.2 町民との交流

| 交流タイプ | 内容 | 報酬 |
|-----------|------|------|
| 日常会話 | 話しかけると英語で返してくる | 小アイテム・好感度UP |
| ミッション依頼 | 「〇〇を作ってほしい」と依頼してくる | 素材・Word Coins |
| 物々交換 | 「コレと交換しない？」と英語で持ちかける | 希少素材 |
| 特技披露 | 好感度が高い町民が特技を見せてくれる | 新レシピ（クラフト解禁） |
| 文化紹介 | 旅人型NPCが各ゾーンの英語表現を教えてくれる | 新パターン解禁ヒント |

### 5.3 町民の属性・種類

各ゾーンの英語パターンに対応した職業・キャラクターを持つ。

| ゾーン | 典型的な町民 | 得意な英語 |
|--------|------------|-----------|
| Zone 1（村） | 農家・先生・子ども | 自己紹介・感情・日常会話 |
| Zone 2（市場） | 商人・医者・料理人 | 提案・依頼・買い物 |
| Zone 3（港） | 船乗り・旅行者・案内人 | 道案内・予約・旅行表現 |
| Zone 4（ビジネス） | ビジネスマン・弁護士 | ビジネス英語・交渉 |
| Zone 5（国際） | 外交官・ジャーナリスト | 国際的なコミュニケーション |

## 6. マルチプレイ設計

### 6.1 最大4人の協力開発

| 機能 | 内容 |
|------|------|
| 共同開発 | 同じ土地に最大4人で同時アクセス。リアルタイムで整地・クラフト・配置が可能 |
| 役割分担 | 採掘・農業・建設・接客など自然に役割が生まれる設計 |
| 共有倉庫 | 素材を共有倉庫に入れてチームで管理 |
| 英語チャレンジ協力 | 一人が答えられないFlash Outputを別のメンバーが助ける「ヘルプ機能」 |
| 街の共同所有 | マルチプレイで作った街は全員が「住人」として持つ。ホストが招待管理 |

### 6.2 ゲームシェア（招待リンク方式）

```
プレイヤーが「友達を招待」ボタンを押す
      ↓
専用招待リンク（URL）が生成される
      ↓
LINEやSNSで友達に送る
      ↓
友達がリンクをタップ
      ↓
Robloxアプリが未インストールの場合 → インストール画面へ
Robloxアカウント未登録の場合 → ワンタップ登録画面へ（無料）
      ↓
登録完了 → 自動的にその街のゲームに参加
      ↓
初回参加者限定で「助っ人ボーナス素材」プレゼント
```

### 6.3 訪問機能（非同期）

- 自分がオフラインの間も友達が街を訪問できる
- 友達が訪問するとMowiが「〇〇ちゃんが来てたよ」と教えてくれる
- 友達の足跡・メモが街の掲示板に残る（英語メッセージ）
- 訪問者が増えるほど「人気の街ランキング」が上昇

## 7. VRロードマップ

### 7.1 Roblox × Meta Quest 対応

RobloxはMeta Quest 2 / 3 / Proに公式対応済み。同じゲームコードがVRでもプレイできる。

| 段階 | プラットフォーム | 主な対応作業 |
|------|---------------|------------|
| Phase 1 | PC / スマホ（Roblox） | 3D空間設計。VR-first UIを意識した設計 |
| Phase 2 | Meta Quest（Roblox） | VR操作UX調整・コントローラー対応 |
| Phase 3 | WebXR（VR Academy） | Robloxから独立したWebXR版に移植 |

### 7.2 VR設計で今から意識すること（Phase 1から）

**UI配置**
- HUD（問題表示・素材バー）は視野角内に自然に収まる設計
- 2Dフラットなメニューより「空間に浮かぶパネル」を基本とする

**Flash OutputのVR入力**
- テキスト入力（キーボード）→ VRでは難しい
- 選択式（コントローラーで指さし選択）をメインに設計
- 音声入力はPhase 2以降で追加（Robloxの音声API活用）

**Mowiの3D存在感**
- VRではMowiがより「実在する」感覚になる
- 肩・手のひら・目の前など、Mowiの居場所をVR空間で自然に設計しておく

### 7.3 Meta Quest でのプレイフロー（Phase 2目標）

```
Meta Quest でRobloxアプリを起動
      ↓
MoWISE Worldを選択
      ↓
VR空間で荒れた世界に降り立つ
      ↓
Mowiが目の前に現れる（手の届く距離）
      ↓
コントローラーで土地を整地する（直感的な操作）
      ↓
空中に浮かぶFlash Outputパネルに答える
      ↓
素材が手元に飛んでくる演出
      ↓
建物を掴んで配置する（直感的なクラフト体験）
```

## 10. Mowiの演出（3D空間版）

### 10.1 存在設計

| 状態 | Roblox内での表現 |
|------|----------------|
| 通常 | プレイヤーの隣を浮遊。ゆっくりした弧を描く |
| 高習熟（★3-5多数） | 明るく輝く。リングが速く回転 |
| 低習熟（全体的に★1-2） | 少し薄暗い。リングがゆっくり |

| トリガー | 演出 |
|---------|------|
| パターン★5到達 | Mowiが空高く飛び上がり、光の柱が立つ |

| 放置後復帰 | 「おかえり」の言葉と共に小さな光が舞う |
