## 1. プロジェクト概要

### 1.1 アプリ名・ブランド

| 項目 | 内容 |
|------|------|
| **アプリ名** | MoWISE（モワイズ） |
| **サブタイトル** | もっと賢く、英語と生きる |
| **ブランド** | WISE English Club |
| **ロゴ** | 虹色グラデーション「MoWISE」テキスト ＋ Mowiリングアイコン |

### 1.2 コンセプト一文

> **「先生とつながるポケモン式英語習得システム」**
> DuolingoでもRPGゲームでもなく、先生（ジムリーダー）が導入し、生徒（トレーナー）がパターン（ポケモン）を捕まえて育てて使う英語学習アプリ。Mowiはユーザーの英語力そのものを映す鏡。

### 1.3 競合との差別化

| 比較軸 | Duolingo | MoWISE |
|--------|----------|--------|
| 先生の関与 | 不可 | ✅ コア機能（ジムリーダー） |
| キャラクター | 外からの応援者 | ✅ ユーザー自身の英語力の鏡 |
| アウトプット設計 | 選択式中心 | ✅ 5層ドリル＋Flash Output必須 |
| 反復練習の動機 | XP・ストリーク | ✅ 自分の変化への気づき |
| 音声学習 | 基本TTS | ✅ リンキング・リダクション対応 |
| AI補強 | なし | ✅ 弱点自動補強 |
| バトル | なし | ✅ 生徒間・AI・レイド対戦 |
| 認定システム | なし | ✅ 先生によるジムバッジ発行 |

### 1.4 ターゲットユーザー

- **プライマリ：** 教室に通う小中学生（英語レベル問わず）
- **セカンダリ：** 英語塾・英語教室の先生
- **第三層：** 独学で英語を学びたい社会人

## 2. コアコンセプト：ポケモンモデル

### 2.1 ポケモンとのマッピング

| ポケモン概念 | MoWISEへの対応 |
|-------------|---------------|
| ポケモンを捕まえる | パターンに「初めて出会う」 |
| レベルを上げる | 5層ドリルで習熟度★を上げる |
| 技を覚える | パターンの使い方バリエーション習得 |
| 進化させる | ★5到達で「進化形（応用表現）」が解禁 |
| バトルで使う | 実際の場面・他の生徒との対戦 |
| 図鑑を埋める | パターンコレクションを完成させる |
| ジムバッジ | 先生が認定するレベルクリア証明 |
| ジムリーダー | 先生（クラス担当教師） |
| トレーナー | 生徒 |

### 2.2 「捕まえる → 育てる → 使って勝つ」の流れ

```
1. 出会う（Discover）
   └── 先生が今週の課題パターンを指定
       └── 図鑑に新パターンが追加される

2. 育てる（Train）
   ├── Layer 0：Sound Foundation（音を聞く）
   ├── Layer 1：Echo Drill（まねる）
   ├── Layer 2：Pattern Quiz + Grammar Code（理解する）
   ├── Layer 3：Flash Output（瞬間英作文）← 最重要
   └── Layer 4：Scene Challenge（実践する）
   → 5層完了で★5 → 進化形が解禁

3. 使って勝つ（Battle）
   ├── AI対戦（デイリーバトル）
   ├── 友達対戦（クラスバトル）
   └── レイドバトル（クラス全員参加）

4. 認定される（Certified）
   └── 先生がジムバッジを発行
       └── 次のエリアが解禁
```

## 3. メインキャラクター：Mowi（モゥィー）

### 3.1 キャラクター設定

| 項目 | 内容 |
|------|------|
| **名前** | Mowi（モゥィー） |
| **外見** | 白く丸いフォルム（宇宙オーブ）、虹色の光輪（リング）、翡翠色の大きな瞳、ピンクの頬 |
| **コンセプト** | 宇宙を漂う知恵の存在。英語パターンのエネルギーを吸収して成長する |
| **役割** | **ユーザーの英語力そのものを映す鏡**。学習の伴奏者・感情フィードバック担当 |
| **声** | 高めで温かみのある効果音（言語なし） |

### 3.2 Mowiの本質：「ユーザーの英語力そのもの」

**これがMoWISEの最重要設計思想。**

- Mowiが輝いているとき＝ユーザーが英語を話せているとき
- Mowiが暗いとき＝ユーザーの英語が錆びついているとき
- キャラクターへの愛着ではなく、**自分への愛着**が生まれる設計

| 要素 | 意味 |
|------|------|
| Mowiが輝く | あなたが話せるようになった証拠 |
| Mowiが暗くなる | あなたの英語が錆びついている |
| リングが増える | 使えるパターンの幅が広がった可視化 |
| Mowiが眠る | 英語力が眠っている状態 |
| Mowiがモノクロ | まだ英語と出会っていない自分 |

### 3.3 成長ステージ

| レベル | 状態 | 見た目 |
|--------|------|--------|
| Lv.1 | 種 | モノクロ、光輪なし |
| Lv.5 | 芽吹き | うっすら色づく |
| Lv.10 | 開花 | 虹色の光輪が現れる |
| Lv.20 | 輝き | フルカラー、光輪がキラキラ |
| Lv.30 | 宇宙 | 星のエフェクトを纏う |

**リングの設計：**
- WISEロゴの虹色（赤→オレンジ→黄→緑→青→紫）を直接継承
- 3本のリングが異なる軌道で回り続ける
- レベルアップ時に新しいリングが生まれるアニメーション
- 目の輝きが学習進捗で増す（左目：シアン→ブルー、右目：グリーン→ティール）

### 3.4 感情ステートとセリフ（場面別ハイブリッド）

**設計ルール（v3.1 改訂）：**
- 練習の流れを邪魔しない場面では **静かな内なる声**
- ユーザーの節目（達成・チェックイン・離脱・復帰・セッション終了）では **Mowiが直接語りかける**
- 「すごい！」「えらい！」「頑張って！」などのDuolingo的応援は禁止
- 基準：**「Mowiがそこで見ていたから言える、具体的な一言」**
- 短く、説明しない。主語はできるだけなし

| 場面 | スタイル | 例 |
|------|---------|-----|
| 練習中（問題回答中） | 静か・内なる声 | 「…出てくる。もう少し。」 |
| 正解・コンボ継続 | 短い語りかけ | 「言えてたよ、さっきの。」 |
| ★アップ・レベルアップ | 直接語りかけ | 「また一段、遠くが見えるようになったね。」 |
| チェックイン前後 | 直接語りかけ | 「今日はどうだった？」「昨日より声が出てたよ。」 |
| 3日以上未ログイン | 直接語りかけ | 「久しぶり。少し遊びたいな、一緒に戻ろう。」 |
| セッション開始 | 静かな語りかけ | 「始めよう。」 |
| セッション終了 | 直接語りかけ | 「今日ここまで来た。明日、続きを見せてね。」 |

**Idle（通常・浮遊）** — 内なる声
アニメーション：ゆっくりとした浮遊
> 「今日も、ここにいる。」
> 「使われるのを、待っている。」
> 「言葉は、体の中にある。」

**Cheer（問題開始・励まし）** — 内なる声
アニメーション：虹をパタパタ振る
> 「言ってみる。」
> 「知ってる。出てくるはず。」
> 「ゆっくりでいい。でも、言う。」

**Think（ヒント提示）** — 内なる声
アニメーション：目がぐるぐる
> 「どこかにある。探してる。」
> 「聞いたことがある、この形。」

**Joy（正解・コンボ継続）** — 語りかけ
アニメーション：ぴょんと跳ねる、虹が広がる
> 「言えてたよ、さっきの。」
> 「今の、ちゃんと繋がった。」
> 「また一つ、遠くまで届いたね。」
> 「体に入ったの、わかる。」

**Sad（不正解）** — 内なる声
アニメーション：うなだれる、色がくすむ
> 「あと少し、だった。」
> 「まだ、体になじんでない。」
> 「もう一回。今度は出る。」

**Grow（★アップ・バッジ取得）** — 語りかけ
アニメーション：輝く光エフェクト
> 「また一段、遠くが見えるようになったね。」
> 「前より広い場所に立ってるよ。」
> 「リングが増えた。見える？」
> 「ここまで来たのは、毎日の分だよ。」

**Sleep（3日以上未ログイン）** — 語りかけ
アニメーション：ぐっすり眠る → 目覚め
> 「久しぶり。少し遊びたいな、一緒に戻ろう。」
> 「3日分、静かだった。声を聞きたかった。」
> 「錆びてはいない。ただ、眠ってただけ。」

**SessionStart（セッション開始）** — 静かな語りかけ
> 「始めよう。」

**SessionEnd（セッション終了）** — 語りかけ
> 「今日ここまで来た。明日、続きを見せてね。」
> 「今日やった分、ちゃんと残ってるよ。」

**CheckinMorning（朝チェックイン後）** — 語りかけ
> 「今日はどんな感じか、教えて。」
> 「声を聞かせて。」

**CheckinEvening（夜チェックイン後）** — 語りかけ
> 「昨日より声が出てたよ。」
> 「今日の分、ちゃんとここにある。」

**Streak（連続ログイン継続中）** — 語りかけ
> 「毎日ここにいるの、知ってるよ。それだけで違う。」

**FirstPattern（初めてのパターン習得）** — 語りかけ
> 「初めて会ったね、この言葉と。」

## 4. チェックイン日記システム

### 4.1 設計思想

**「伴奏型日記」として設計する。**

Mowiのセリフを一方的に表示するのではなく、ユーザーが自分の状態を選択することで内なる声が浸透する仕組み。人は受け取った言葉より、自分が選んだ言葉の方が深く残る。

### 4.2 毎日のチェックインフロー

**練習前チェックイン（30秒）**
```
「今日の英語、どんな感じ？」

選択肢：
・自信ある
・まあまあ
・不安
・わからない
```
→ 選択に応じてMowiの表情が変わる
→ 「今日の一言」が決まる（Mowiの内なる声）

**練習後チェックイン（30秒）**
```
「やってみてどうだった？」

選択肢：
・言えた気がする
・まだしっくりこない
・思ったより難しかった
・楽しかった
```
→ 朝との差分がMowiに反映される
→ 今日の記録として保存される

### 4.3 記録の蓄積と振り返り

- 選んだ言葉がMowiの「今日の一言」として画面に残る
- 毎日の積み重ねが記録として蓄積
- 週次・月次で過去の記録を振り返れる
- 3ヶ月後に「不安」が「言えた」に変わっている＝英語習得の証拠

### 4.4 先生との連動

- 生徒のチェックインデータが先生ダッシュボードに反映
- 「不安」が続く生徒を先生が早期発見できる
- 個別介入のタイミングを先生が判断しやすくなる

## 5. ゲーム中毒ループ設計

### 5.1 設計の核心思想

**「やりたくなる」ではなく「やらないと自分が気になる」設計。**
Mowi＝自分の英語力の鏡、という前提で全メカニクスが機能する。

### 5.2 1日のコアループ

```
アプリを開く
    ↓
【朝のチェックイン】30秒
「今日の英語、どんな感じ？」
→ 選択 → Mowiの表情変化 → 今日の一言決定
    ↓
【練習】5〜15分（後述）
    ↓
【夜のチェックイン】30秒
「やってみてどうだった？」
→ 選択 → 朝との差分がMowiに反映 → 記録保存
    ↓
「明日、続きが見える」を予告して終わる
```

### 5.3 練習中の3層中毒メカニクス

**Layer 1：入口は必ず1問から**
- 最初の問題は1問だけ
- 答えた瞬間に次が滑り込む。止まるタイミングを作らない
- Mowiの反応（自分の英語力の状態）が見たくて続けてしまう

**Layer 2：コンボが「切れかけ」になる瞬間を作る**
```
コンボ継続中   → Mowiがどんどん輝いていく
コンボ切れかけ → Mowiの光がリアルタイムで薄れていく（視覚的）
コンボが切れた → Mowiが一瞬暗くなる
               → 「もう一回」「今日はここまで」をユーザーが選ぶ
```
コンボを切らしたくない＝Mowiを暗くしたくない＝自分の英語力を落としたくない、という感覚に直結する。

**Layer 3：終わり際に「続きが見える」**
```
セッション終了時：
「P007まで来た。あと3問でP010の進化形が解禁される。」
Mowiが「明日、見せてあげる。」と言う
```
「明日」という言葉を残して終わる。翌朝開く理由になる。

### 5.4 週次ループ

```
月曜：Mowiの今週のスタート状態が記録される
毎日：練習するたびにMowiが少しずつ変化
日曜：今週のMowiと先週のMowiを並べて表示
     「先週より、遠くが見えるようになった。」
```

1週間の自分の変化が視覚化されることが継続の最大の動機になる。

### 5.5 中毒性の設計原則まとめ

| やめられない理由 | 仕組み |
|----------------|--------|
| 自分が気になる | Mowi＝自分の鏡なので状態が気になって開く |
| 切りたくない | コンボ＝Mowiの輝き、切れかけが見えるから続ける |
| 明日が気になる | 終わり際に「続き」を予告して閉じる |
| 記録が残る | 日記として蓄積され、過去の自分と比べられる |
| 週単位で変化が見える | 先週のMowiとの比較で成長が可視化される |

## 6. 全体連動設計

### 6.1 一本の線で表すコンセプト

**「自分の英語力が、見える。育つ。証明される。」**

### 6.2 最重要連動ポイント

```
【ゲームの結果】→【Mowiの変化】→【チェックインの問い】
      ↑                                    ↓
      └──────【また練習したくなる】←────────┘
```

この円環が回り続けることが「英語習得と中毒性が一致している状態」。

### 6.3 データの流れ

```
ユーザーが練習する
    ↓
正誤・速度データ → AI弱点補強システム
    ↓                    ↓
★習熟度が上がる    弱点パターンをデイリーに自動追加
    ↓
Mowiが輝く
    ↓
チェックインで自己評価
    ↓
日記として蓄積
    ↓
先生ダッシュボードへ流れる
    ↓
先生がバッジ承認 or アラート送信
    ↓
生徒のMowiが反応
    ↓
また練習したくなる
```

### 6.4 各要素の役割と連動先

**Mowi**
- 入力：練習結果 ＋ チェックイン選択 ＋ ストリーク日数
- 出力：表情・輝き・内なる声（一言）
- 連動：ゲームループ（コンボで変化）／チェックイン日記（選択がMowiに反映）／週次レポート

**チェックイン日記**
- 入力：ユーザーの自己評価（選択式）
- 出力：Mowiの今日の一言 ＋ 記録
- 連動：Mowi（状態に反映）／先生ダッシュボード／週次ループ

**ゲームループ**
- 入力：パターン練習の正誤・速度
- 出力：★習熟度 ＋ XP ＋ コンボ
- 連動：Mowi（コンボで輝き変化）／パターン図鑑（★が上がる）／AI弱点補強／先生ダッシュボード

**パターン図鑑**
- 入力：★習熟度の蓄積
- 出力：進化形パターン解禁 ＋ ジムバッジ申請権
- 連動：ゲームループ／先生ダッシュボード／Mowi成長（全体★平均がLvに反映）

**先生ダッシュボード**
- 入力：全生徒の習熟度・チェックイン・バトルデータ
- 出力：課題設定 ＋ バッジ承認 ＋ アラート ＋ レイド設定
- 連動：パターン図鑑（課題が反映）／バトル／生徒のMowi

**バトル**
- 入力：習得済みパターン（★1以上）
- 出力：XP ＋ ランキング ＋ クラス一体感
- 連動：ゲームループ／先生ダッシュボード／Mowi

## 7. 学習システム：5層ドリル構造

### 7.1 設計原則（SLA研究準拠）

全パターンはLayer 0〜4を順番にクリアすることで★5（完全習得）になる。各Layerは必ずインプットとアウトプットをペアで実施する。

| 方法 | 科学的根拠 | MoWISEでの実装 |
|------|-----------|---------------|
| 間隔反復 | 忘却曲線に基づく最適タイミング（Ebbinghaus, 1885） | ★の時間減衰 ＋ AI自動再出題 |
| 強制アウトプット | 5週間実施で流暢さ・正確さが有意向上（Swain, 1985） | Flash Output（制限時間つき瞬間英作文） |
| インタラクション | 意味交渉を含む対話が最も強い習得効果 | Scene Challenge ＋ バトルシステム |

### 7.2 Layer 0：Sound Foundation（★0→1）

| 項目 | 詳細 |
|------|------|
| 目的 | フォニックス・リンキング・フラッピング・リダクションを体に入れる |
| インプット | 通常発音 vs ネイティブ発音の聞き比べ → Y/N |
| アウトプット | 音声変化を自分で発音→録音→波形を視覚化して比較（Phase 2） |
| 例題 | 「What do you want to eat?」 vs「Whaddya wanna eat?」 |
| 所要時間 | 1〜2分 |
| 技術 | ElevenLabs TTS ＋ Howler.js ＋ WebAudio API |

### 7.3 Layer 1：Echo Drill（★1→2）

| 項目 | 詳細 |
|------|------|
| 目的 | パターンの音・リズムを体に入れる |
| インプット | 音声を聞いて3択から正解を選ぶ（Sound Match） |
| アウトプット | 音声再生後1秒以内に発音→録音→お手本波形と並べて視覚比較 |
| ゲーム要素 | 波形の一致率80%以上でコンボボーナス。Mowiが反応 |
| 所要時間 | 2〜3分 |

### 7.4 Layer 2：Pattern Quiz + Grammar Code（★2→3）

| 項目 | 詳細 |
|------|------|
| 目的 | パターンの意味・構造・文法を完全理解する |
| インプット | 文脈付き4択クイズ（日本語→英語 / 英語→日本語） |
| アウトプット | 英文の各単語を品詞別にタップして色を塗り分け（Grammar Code） |
| Grammar Code例 | 「I like big cats.」→ I(青) like(赤) big(緑) cats(青) を自分でタップ |
| 所要時間 | 2〜3分 |

### 7.5 Layer 3：Flash Output（★3→4）← **MVP最重要・必須**

> 【なぜ最重要か】SLA研究で最も効果が実証されているPushed Outputの実装。日→英の反射訓練なしに英語は口から出ない。

| 項目 | 詳細 |
|------|------|
| 目的 | 日本語を見て即座に英文を作る「瞬間英作文」の反射回路を作る |
| 難易度1（初級） | タイル選択式：バラバラになった単語タイルを並べる（制限時間8秒） |
| 難易度2（中級） | キーボード入力式：日本語を見てテキストで英文を入力（制限時間10秒） |
| 難易度3（上級） | 音声入力式：マイクに向かって英語で発話（Phase 2） |
| コンボシステム | 連続正解でMowiが輝く。3連続でXPボーナス、5連続で特別演出 |
| 所要時間 | 3〜5分 |
| **Mowi連動** | コンボ継続中はMowiが輝き続ける。切れかけると光が薄れる（視覚） |

### 7.6 Layer 4：Scene Challenge（★4→5）

| 項目 | 詳細 |
|------|------|
| 目的 | リアルな会話場面でパターンを実際に使えるか確認する |
| 形式 | 会話シナリオ選択式 / 穴埋め入力。5ターン完走でクリア |
| 発話式（Phase 2） | NPCが英語で話しかける → マイクで英語で返す |
| 解禁条件 | Layer 3（Flash Output）完了後 |
| クリア条件 | ★5到達 → 進化形パターン解禁 |
| 所要時間 | 3〜5分 |

## 9. バトルシステム

### 9.1 デイリーバトル（AI対戦）

- 毎日リセット（0時）
- 習得済みパターン（★1以上）のみで戦う
- 難易度は現在の★平均から自動調整
- 勝利でXP ＋ ゴールド獲得
- Mowiが勝利時に特別反応

### 9.2 クラスバトル（生徒間対戦）

- 先生が設定したパターン範囲内で対戦
- 非同期対戦（相手の時間に縛られない）
- 同じパターンセットで互いにスコアを競う
- 週次でランキングリセット

### 9.3 レイドバトル（クラス全員参加イベント）

- 先生が週1で設定
- クラス全員で協力してボスパターンを攻略
- 全員の正解数を合計してボスのHPを削る
- 成功でクラス全員に特別バッジ ＋ XP
- Mowiが「チームで英語力が届いた」演出をする

### 9.4 バトルの勝敗ルール

- ★0のパターンでは戦えない → 学習の必然性
- 習熟度★が高いほど攻撃力アップ
- 制限時間内の回答速度もスコアに影響

## 13. ゲーミフィケーション要素

### 13.1 習熟度システム

- 各パターンに個別の習熟度★（0〜5）
- ★5 = 完全習得 → 進化形パターン解禁
- ★は時間経過（未練習）で減衰する（忘却曲線）
- 全体の平均★がMowiのLvに反映

### 13.2 コンボシステム（Mowi連動）

```
1連続：通常
3連続：Mowiの輝きが増す「COMBO x3!」
5連続：Mowiがぴょんと跳ねる
7連続：「FEVER!」+ BGMテンポ上昇
10連続：虹色エフェクト「LEGENDARY!」

切れかけ演出：
- 回答が遅いとMowiの光がリアルタイムで薄れる
- 光が薄れるのを見て「切らしたくない」と焦る
- これが「あと1問だけ」を生む
```

### 13.3 ジムバッジ

- エリアクリアで先生に申請
- 先生が確認・承認して正式発行
- バッジは図鑑に恒久表示
- バッジ取得でXP大量獲得・次エリア解禁
- **バッジ取得時はMowiが最高輝度になる**

### 13.4 ストリーク

- 連続ログイン日数を記録
- 3日・7日・30日でMowiが特別アニメーション
- ストリーク途切れでMowiが眠る（復帰を促す）

### 13.5 週次振り返り

- 毎週日曜：今週のMowiと先週のMowiを並べて表示
- 「先週より、遠くが見えるようになった。」
- チェックイン日記の週次サマリーも表示

### 13.6 XP・レベル・ランキング

- 正解・バトル勝利・ストリーク・チェックインでXP獲得
- XPでユーザーレベルアップ → Mowiも成長
- クラス内週次ランキング（XP順）
- バトル勝率ランキング
