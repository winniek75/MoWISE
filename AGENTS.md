## 3. 絶対にブレてはいけないコアコンセプト

以下の6項目は全タスクにおいて最優先で守ること。

1. **Mowi = ユーザーの英語力そのもの**
   - 外からの応援キャラ・マスコットではない
   - 「内なる鏡」。生徒の英語力が上がるほどMowiが変化する

2. **チェックイン日記 = 差別化の核心**
   - Duolingoとの最大の違い
   - 内発的動機設計。外部報酬（XP・ストリーク）だけに頼らない

3. **先生（ジムリーダー）がいることで学習が成立する**
   - 教室 × アプリの二重設計が前提
   - 先生なしのソロ学習は補助的な位置づけ

4. **Mowiのセリフは「場面別ハイブリッド」**
   - 練習中（問題回答中 / Cheer / Think）→ 静かな内なる声スタイルを維持
   - 正解・コンボ / ★アップ / チェックイン / ログイン復帰 / セッション終了 → Mowiが直接語りかけるスタイル
   - ❌ NG例：「すごい！」「えらい！」「頑張って！」（Duolingo的応援禁止）
   - ✅ OK例：「言えてたよ、さっきの。」「また一段、遠くが見えるようになったね。」
   - 基準：**「Mowiがそこで見ていたから言える、具体的な一言」**にする

5. **ストーリー（RPG村人会話）は廃止済み**
   - v3.0で正式廃止。蒸し返さないこと
   - ゲーム性はバトルシステムとパターン図鑑で担保する

6. **成長メタファーは宇宙オーブ（Lv.1〜30）**
   - 植物メタファー（種→芽→開花）は廃止済み
   - ビジュアルは虹色グラデーションのオーブ形状

## 4. 開発方針

### やること

- Vue 3 + Capacitor を唯一のフロントエンド基盤とする（React/React Native 導入禁止）
- 既存の Vue.js コード資産を最大限活用する
- Supabase を BaaS として一本化する（Firebase 等への移行検討禁止）
- 学習設計は SLA（第二言語習得）研究の知見に基づく
- Layer 2 は「Pattern Sense（スロット穴埋め＋音声予測）」として実装する（Grammar Code は廃止）

### やらないこと（禁止）

- ❌ React / React Native / Next.js の導入
- ❌ ElevenLabs の TTS 使用（Google Cloud TTS Neural2 に移行済み）
- ❌ Grammar Code（品詞色分けタップ）の復活
- ❌ RPG村人ストーリーの復活
- ❌ 植物メタファーの使用
- ❌ Mowiを外部応援キャラとして描写

### 不正解時UX の原則（v3.1 決定）

知識より感覚を先に教える：
1. まず「不自然な音」を聞かせる
2. Mowiが「…なんか変」と反応する
3. ユーザーが任意で「なぜ違うか」の説明を見る

## 10. Robloxゲームコンセプト全面改訂（2026年3月12日）

> **⚠️ Roblox関連タスクでは必ずこのセクションを確認してください。**
> v1.0（タイクーン設計書）の内容は廃止。v2.0（サンドボックス設計書）が唯一の正ソースです。

### 10.1 改訂の核心

| 項目 | v1.0（廃止） | v2.0（現行） |
|------|------------|------------|
| ゲームジャンル | タイクーン（既製品をCoinsで購入・建設） | **サンドボックス（素材収集→自由クラフト）** |
| 世界観 | 何もない空き地から開発 | **荒れ果てた廃墟の世界から文明を復興** |
| Mowiの立ち位置 | 街の上空に浮かぶ「魂」 | **プレイヤーと並走する相棒・メンター** |
| 建設方式 | Coinsで建物を購入 | **素材を集めて自由にクラフト・配置** |
| 経済設計 | Word Coins（単一通貨） | **素材（Materials）＋Word Coins（併用）** |
| 町民の出現 | 建物建設で自動配置 | **徐々に移住してくる。英語で会話してアイテム交換** |
| 農業・商売 | なし | **畑・お店・商売システムあり** |
| マルチプレイ | 友達訪問・バトル中心 | **最大4人で同じ土地を協力開発** |
| ゲームシェア | 未設計 | **招待リンクでRoblox登録＆即参加** |
| VR対応 | Phase 3に言及のみ | **Roblox → Meta Quest → WebXR の段階ロードマップ確定** |

### 10.2 引き継ぐ資産（v1.0から変更なし）

以下はv2.0でも完全に活用する。実装時に再設計は不要。

- ✅ 5ゾーン×99建物のマップ構造（世界観を「廃墟→復興」に再解釈して活用）
- ✅ パターン★と建物・エリア解禁のリンク設計
- ✅ Flash Output → リソース獲得のコアループ
- ✅ NPC会話ミッション（Scene Challenge相当）
- ✅ WebアプリとのAPI連動設計（R4設計書そのまま）
- ✅ チャートマスター・Classroom Mode
- ✅ English Obstacle Course、Mowi Café Time、Living Town Map の3メカニクス

### 10.3 Mowiの新定義（Roblox v2.0限定）

Webアプリでの「内なる鏡」原則は変わらない。Roblox内での表現方法が変わる。

- **Webアプリ**：画面の中で輝くオーブ。プレイヤーの英語力の反映
- **Roblox v2.0**：プレイヤーのアバターの隣を飛ぶ存在。探索・クラフト・会話を共にする

**Mowiがやってはいけないこと（Roblox v2.0）**
- ❌ プレイヤーに媚びる・褒めすぎる（Duolingo化禁止）
- ❌ ペットとして描写する（依存関係を作らない）
- ❌ 英語力と無関係に変化する（Mowiの状態は常に習熟度の鏡）

### 10.4 素材×英語パターンのリンク原則

英語パターンの「意味・文脈」と獲得素材を対応させる。

| 英語パターンの性質 | 対応素材 |
|-----------------|---------|
| 自己紹介・存在表現（I'm / This is） | 石材・木材（基礎素材） |
| 好みの表現（I like） | 花・装飾素材 |
| 欲求・必要の表現（I want / I need） | 食料素材 |
| 所有の表現（I have） | 倉庫拡張素材 |
| 能力の表現（I can） | 工具素材 |
| 丁寧な依頼（I'd like ... please） | 高級建材 |
| 複雑・長文パターン（P050〜） | レア素材・鉱石 |

### 10.5 VRロードマップ（確定）

```
Phase 1：Roblox（PC / スマホ）で3D街復興ゲームをリリース
Phase 2：Meta Quest対応（Robloxの設定変更＋VR用UX調整）
         ※ RobloxはMeta Quest 2/3/Pro公式対応済み。移植作業ほぼ不要
Phase 3：WebXR（VR Academy）への本格移植
```

**Phase 1から意識すること（実装上の制約）：**
- HUDは「空間に浮かぶパネル」方式。2DフラットなメニューUI禁止
- Flash OutputはVRでも機能する「選択式」をメインで設計
- Mowiの居場所（肩・手のひら・目の前）をVR空間で自然に成立するよう設計
